创建帐户

创建一个维护帐户“sysman”，把它加入到管理群组“admin”。

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 创建管理群组“admin”。

    groupadd admin

  3. 创建帐户“sysman”。

    useradd -g admin -m -d /home/sysman -s /bin/bash sysman

    表示创建一个名为“sysman”的用户，属于“admin”用户组，家目录为“/home/sysman”。

    参数说明如下：

    -g：指定维护帐户群组为“admin”。

    -m：如果用户的家目录不存在，则建立指定的家目录“/home/sysman”，如果已经存在，则此参数无效。

    -d：指定用户的家目录。

    -s：指定用户的shell类型。

  4. 设置“sysman”帐户的密码。

    passwd sysman

    当出现如下信息时，表示密码设置成功。
    Changing password for sysman.

修改密码（命令行方式）

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 修改root密码。出于帐号安全考虑，建议首次登录必须修改默认密码，并定期维护密码信息。
    passwd root
    Changing password for user root.
New password:    //输入新密码，密码规则请参考密码维护建议。
Retype new password:    //确认新密码
passwd: all authentication tokens updated successfully.

    新密码在下次登录时生效。

  3. 参考上述步骤，修改admin用户密码。

    passwd admin

  4. 参考上述步骤，修改ivsoper用户密码。

    passwd ivsoper

删除帐户

对于因临时需要而创建的维护帐户，在使用后请删除该帐户。

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 删除用户sysman。

    userdel sysman -r

锁定帐户

如果维护帐户暂时不需使用，可锁定该帐户，等需要使用时再解锁。

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 锁定帐户“sysman”。

    passwd -l sysman

    当出现如下信息时，表示锁定帐户成功。
    Locking password for user test.
passwd: Success

解锁帐户

如果维护帐户被锁定后需要再次使用，可以执行解锁帐户操作。

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 解锁帐户“sysman”。

    passwd -u sysman

    当出现如下信息时，表示解锁帐户成功。
    Unlocking password for user test.
passwd: Success

设置帐户有效期

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 设置“sysman”帐户的密码有效期为90天，并在到期前7天提醒修改密码。

    passwd -x 90 -w 7 sysman

    当出现如下信息时，表示设置帐户的有效期成功。
    Adjusting aging data for user test.
passwd: Success

帐户安防和审计

建议您定期检查维护帐户的密码有效期，根据需要及时更新即将过期的帐户密码，帐户密码通常90天更新一次。

  1. 以root用户登录操作系统（ 如何通过网口登录操作系统）。
  2. 查询帐户的密码有效期。

    passwd -S sysman

    查询结果显示格式：
    name status mm/dd/yy min max warn
sysman PS 2020-01-14 0 90 7 35 (Password set, SHA512 crypt.)

    主要参数说明如下：

    • sysman：帐户名称。
    • status：帐户状态，包括：PS（passworded）、LK（locked）和NP（no）。
    • 2020-01-14：当前日期。
    • 0：帐户必须经多少天才可以更改口令。
    • 90：帐户口令经多少天到期。
    • 7：帐户口令到期之前通知帐户更改口令的天数。

    如果某个帐号没有密码，请立即设置密码。
